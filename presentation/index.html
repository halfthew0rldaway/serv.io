Latar Belakang Masalah

  • Pencatatan data servis dan riwayat perangkat masih dilakukan secara manual menggunakan buku tulis atau spreadsheet yang tidak terintegrasi.
  • Pelanggan kesulitan melacak proses perbaikan perangkat mereka secara mandiri karena tidak ada sistem tracking yang memadai.
  • Proses kalkulasi harga, penggunaan suku cadang, dan pembuatan invoice memakan waktu dan rentan akan *human error*.
Solusi:
Membangun Serv.io sebagai platform digital terpusat yang menghubungkan Admin, Teknisi, dan Pelanggan dalam satu ekosistem yang terstruktur.

Teknologi yang Digunakan

Aplikasi ini dibangun menggunakan arsitektur modern Frontend-Backend (Decoupled) untuk menjamin skalabilitas dan performa.

Kategori Teknologi / Framework Fungsi Utama
Frontend React.js (Vite), TailwindCSS Antarmuka pengguna (UI/UX) dan *Client-Side Rendering*
Backend Node.js, Express.js REST API, Logika bisnis, dan Autentikasi JWT
Database MySQL, Prisma ORM Penyimpanan relasional dan query data struktural
Tools Chart.js / Recharts Visualisasi data statistik pada Dashboard Admin

Lacak Status Perbaikan (Tracking) ✔ ✔ ✔ (Via Resi)

Alur Kerja Sistem (Workflow)

  • 1. Pendaftaran (Admin): Pelanggan datang, Admin mendaftarkan perangkat ke sistem. Status menjadi DITERIMA.

  • 2. Diagnosis (Teknisi): Teknisi mengecek kerusakan perangkat, menginput catatan, dan memberikan estimasi biaya. Status menjadi DIDIAGNOSIS.

  • 3. Persetujuan (Admin): Admin menghubungi pelanggan. Jika disetujui, lanjut ke perbaikan. Status menjadi DISETUJUI.

  • 4. Perbaikan (Teknisi): Teknisi memperbaiki perangkat, menambahkan catatan akhir, dan mendata suku cadang yang dipakai. Status menjadi DALAM PERBAIKAN lalu SELESAI.

  • 5. Pengambilan (Admin): Pelanggan mengambil perangkat, Admin mencetak bukti pembayaran (Invoice). Status menjadi DIAMBIL.
